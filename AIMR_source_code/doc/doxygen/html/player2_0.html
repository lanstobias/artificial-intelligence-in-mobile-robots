Player 2.0 driver specification
Name:
  • epuck
Provides:
  • position2d
  • ir
  • camera
  • opaque (for LEDs)
Requires:
  • none
Implemented requests:
interface_position2d
  • PLAYER_POSITION2D_CMD_VEL
  • PLAYER_POSITION2D_CMD_CAR
  • PLAYER_POSITION2D_REQ_GET_GEOM
  • PLAYER_POSITION2D_REQ_SET_ODOM
  • PLAYER_POSITION2D_REQ_RESET_ODOM
  • PLAYER_POSITION2D_DATA_STATE
interface_ir :
  • PLAYER_IR_POSE
  • PLAYER_IR_DATA_RANGES
interface_camera :
  • PLAYER_CAMERA_DATA_STATE
opaque (for LEDs)
  • PLAYER_OPAQUE_CMD
Configuration file example:
driver
(
name "epuck"
plugin "libepuck"
provides ["position2d:0"
"ir:0"
"camera:0"
"ring_leds:::opaque:0"
"front_led:::opaque:1"
"body_led:::opaque:2"]
port "/dev/rfcomm1"
)

Each of the ring, front, and body LEDs can be used independently, and the keys "ring_leds", "front_led" and "body_led" must be placed in LEDs opaque interface. Without them the interface won't be subscribed.
